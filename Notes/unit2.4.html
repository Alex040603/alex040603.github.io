Class

Inheritence

This feature allows you to use the fields of the base class in the sub and superclass

Eg:

Base class: Sport

Sub class: BallSport

Super class: Tennis

Sport may have fields tending to all types of sports. The sub class may use some of those fields such as one that involves a sport with a ball. The super class may use new fields that are made in the sub class.


Defining classes

Base class:

Base classes are made like structures but the "struct" is replaced by "class"

Sub class:

Eg:

class BallSport: Sport {

    *code*

}


Overriding methods and properties

Used to override a field in the base class. For example if the base class has left a field blank, the sub class may override that function to add one of its own.

Used like the following:

class Tennis: Sport {

    override func onGrass() {

        print("It's not played on grass")

    }

}

Just imagine the class Sport had the field "onGrass" that was not used becuase a random sport could or could not be played on grass. The override was used here becuase tennis isnt played on grass, a property not specified by the Sports base class.
